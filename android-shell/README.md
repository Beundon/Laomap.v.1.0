# Android shell notes

This app is a single React/TypeScript codebase. The web build (`npm run build`,
output to `dist/`) is deployed two ways:

1. **Web app**: `dist/` is published directly to GitHub Pages.
2. **Android app**: `dist/` is wrapped by [Capacitor](https://capacitorjs.com)
   into a native Android WebView shell, then compiled to an APK by Gradle in
   GitHub Actions. The Android project lives in `android/` (generated by
   `npx cap add android`, committed to the repo so CI doesn't need to
   regenerate native project files on every run).

## Why this approach

- One codebase, one set of bugs to fix, one design system.
- `navigator.geolocation`, `IndexedDB` (via Dexie), and the File APIs used by
  the import engine all work unmodified inside a Capacitor WebView -- no
  platform-specific branching needed in `src/core/*`.
- Capacitor's native plugin bridge (`@capacitor/geolocation`,
  `@capacitor/filesystem`) is installed and available as a drop-in upgrade
  path if/when higher-precision native GPS or native file pickers are
  needed beyond what the WebView APIs provide. The current boot-detection
  engine (`src/core/bootDetection.ts`) deliberately uses the standard Web
  Geolocation API so it runs identically on claude.ai's web build, a plain
  mobile browser, and the Capacitor-wrapped APK without any `if (isNative)`
  branching.

## Local development workflow

```bash
npm install
npm run build                 # builds dist/
npx cap sync android          # copies dist/ into android/app/src/main/assets
cd android && ./gradlew assembleDebug
# APK at android/app/build/outputs/apk/debug/app-debug.apk
```

## Upgrading to native SQLite storage (optional, future work)

The spec calls for "SQLite/Hive for Mobile" local storage. Because the
mobile build is a WebView, `Dexie`/IndexedDB (the same engine used by the
web build) already persists data natively on-device via the WebView's own
storage sandbox, fully offline, with no extra dependency. If a future
version moves away from a WebView shell (e.g. a fully native rewrite), swap
`src/storage/db.ts` for `@capacitor-community/sqlite` behind the same
exported interface (`StoredLayer`, `StoredTrack`, etc.) so the rest of the
app doesn't need to change.

## Signing release builds

The CI workflow only produces **debug** APKs (unsigned, installable with
`adb install` or sideloading with "unknown sources" enabled). To ship a
signed release build for the Play Store, add a signing keystore as a GitHub
Actions secret and extend the `android` job in
`.github/workflows/deploy.yml` with a `signingConfig` step -- see the
comments in that file for where to plug it in.
